## Allocation Engine

### Purpose
Distribute total capital fairly across accounts based on commission rates while maintaining unpredictability.

### Requirements
1. **Commission-Weighted**: Higher commission accounts get proportionally more
2. **Monotonic**: Higher commission always results in equal or higher allocation
3. **Randomized**: ±15% variance to avoid patterns
4. **Capital-Respecting**: Total never exceeds available capital
5. **Reproducible**: Same seed produces same result

### Algorithm

```python
def allocate(accounts: List[Account], total_capital: float, seed: str) -> List[Allocation]:
    """
    Allocate capital across accounts with commission-weighted monotonic distribution
    
    Args:
        accounts: List of accounts with commission_pct
        total_capital: Total amount to distribute
        seed: Random seed for reproducibility
    
    Returns:
        List of allocations with amounts
    """
    
    # Initialize seeded random number generator
    rng = SeededRandom(seed)
    
    # Step 1: Calculate normalized weights
    total_commission = sum(acc.commission_pct for acc in accounts)
    weights = {
        acc.id: acc.commission_pct / total_commission 
        for acc in accounts
    }
    
    # Step 2: Calculate base allocations
    base_allocations = []
    for account in accounts:
        base_amount = total_capital * weights[account.id]
        base_allocations.append({
            'account_id': account.id,
            'commission': account.commission_pct,
            'base_amount': base_amount
        })
    
    # Step 3: Apply randomization (±15%)
    randomized_allocations = []
    for alloc in base_allocations:
        base = alloc['base_amount']
        variance = base * 0.15
        
        # Generate random offset
        offset = rng.uniform(-variance, variance)
        tentative_amount = base + offset
        
        randomized_allocations.append({
            'account_id': alloc['account_id'],
            'commission': alloc['commission'],
            'tentative_amount': max(tentative_amount, 0.01)  # Min $0.01
        })
    
    # Step 4: Enforce monotonicity
    # Sort by commission (ascending)
    sorted_allocs = sorted(randomized_allocations, key=lambda x: x['commission'])
    
    monotonic_allocations = []
    previous_amount = 0
    
    for alloc in sorted_allocs:
        # Ensure this amount >= previous amount
        enforced_amount = max(alloc['tentative_amount'], previous_amount + 0.01)
        
        monotonic_allocations.append({
            'account_id': alloc['account_id'],
            'amount': enforced_amount
        })
        
        previous_amount = enforced_amount
    
    # Step 5: Normalize to total capital
    actual_total = sum(alloc['amount'] for alloc in monotonic_allocations)
    scaling_factor = total_capital / actual_total
    
    final_allocations = []
    for alloc in monotonic_allocations:
        final_amount = round(alloc['amount'] * scaling_factor, 2)
        final_allocations.append({
            'account_id': alloc['account_id'],
            'amount': final_amount
        })
    
    # Verify total matches (within rounding error)
    final_total = sum(alloc['amount'] for alloc in final_allocations)
    if abs(final_total - total_capital) > 0.10:
        raise AllocationError(f"Total mismatch: {final_total} != {total_capital}")
    
    return final_allocations
```

### Example Execution

**Input:**
```python
accounts = [
    Account(id='A', commission_pct=1.0),
    Account(id='B', commission_pct=3.0),
    Account(id='C', commission_pct=5.0)
]
total_capital = 900.00
seed = "round_12345"
```

**Step-by-Step:**

1. **Calculate Weights:**
   ```
   Total commission = 1.0 + 3.0 + 5.0 = 9.0
   
   Weight_A = 1.0 / 9.0 = 0.111
   Weight_B = 3.0 / 9.0 = 0.333
   Weight_C = 5.0 / 9.0 = 0.556
   ```

2. **Base Allocations:**
   ```
   Base_A = 900 * 0.111 = 100.00
   Base_B = 900 * 0.333 = 300.00
   Base_C = 900 * 0.556 = 500.00
   ```

3. **Apply Randomization (±15%):**
   ```
   Variance_A = 100 * 0.15 = 15.00
   Variance_B = 300 * 0.15 = 45.00
   Variance_C = 500 * 0.15 = 75.00
   
   # Random offsets (example)
   Offset_A = -8.00  →  Tentative_A = 92.00
   Offset_B = +12.00 →  Tentative_B = 312.00
   Offset_C = -22.00 →  Tentative_C = 478.00
   ```

4. **Enforce Monotonicity:**
   ```
   Sort by commission: A(1%), B(3%), C(5%)
   
   Enforced_A = max(92.00, 0 + 0.01) = 92.00
   Enforced_B = max(312.00, 92.00 + 0.01) = 312.00
   Enforced_C = max(478.00, 312.00 + 0.01) = 478.00
   
   Total = 92 + 312 + 478 = 882.00
   ```

5. **Normalize to Total:**
   ```
   Scale = 900 / 882 = 1.0204
   
   Final_A = 92.00 * 1.0204 = 93.88
   Final_B = 312.00 * 1.0204 = 318.37
   Final_C = 478.00 * 1.0204 = 487.75
   
   Total = 93.88 + 318.37 + 487.75 = 900.00 ✓
   ```

**Output:**
```python
[
    {'account_id': 'A', 'amount': 93.88},
    {'account_id': 'B', 'amount': 318.37},
    {'account_id': 'C', 'amount': 487.75}
]
```

### Properties Verified
- ✅ **Commission-weighted**: C (5%) > B (3%) > A (1%)
- ✅ **Monotonic**: 487.75 ≥ 318.37 ≥ 93.88
- ✅ **Randomized**: Not exactly proportional (variance applied)
- ✅ **Capital-respecting**: Total = 900.00 exactly
- ✅ **Reproducible**: Same seed = same result

## IP Conflict Detection

### Purpose
Prevent provider detection by enforcing IP usage rules.

### Rule Types

#### 1. Concurrent Usage Limit
**Rule:** Max N devices using same IP simultaneously

```python
def check_concurrent_limit(ip_address: str, provider: str) -> bool:
    """
    Check if IP exceeds concurrent device limit
    
    Returns:
        True if within limit, False if violation
    """
    
    # Get provider rule
    rule = get_provider_rule(provider)
    max_concurrent = rule.max_concurrent_devices
    
    # Count active devices using this IP
    active_devices = db.query("""
        SELECT COUNT(DISTINCT device_id)
        FROM device_ips
        WHERE ip_address = :ip
          AND is_active = TRUE
    """, ip=ip_address)
    
    return active_devices < max_concurrent
```

**Example:**
```
Rule: max_concurrent = 3
Current devices using 192.168.1.100: [D1, D2, D3]
New device D4 wants to use 192.168.1.100
Result: VIOLATION (3 >= 3)
```

#### 2. Cooldown Period
**Rule:** IP cannot be reused within N seconds after last use

```python
def check_cooldown(ip_address: str, provider: str) -> bool:
    """
    Check if IP is in cooldown period
    
    Returns:
        True if cooldown passed, False if still in cooldown
    """
    
    rule = get_provider_rule(provider)
    cooldown_seconds = rule.cooldown_seconds
    
    # Get last usage time
    last_used = db.query("""
        SELECT MAX(used_at)
        FROM ip_usage_logs
        WHERE device_ip_id IN (
            SELECT id FROM device_ips WHERE ip_address = :ip
        )
        AND provider = :provider
    """, ip=ip_address, provider=provider)
    
    if last_used is None:
        return True  # Never used, no cooldown
    
    elapsed = now() - last_used
    return elapsed >= cooldown_seconds
```

**Example:**
```
Rule: cooldown = 300 seconds (5 minutes)
Last used: 10:00:00
Current time: 10:03:00
Elapsed: 180 seconds
Result: VIOLATION (180 < 300)
```

#### 3. Hourly Usage Limit
**Rule:** IP can only be used N times per hour

```python
def check_hourly_limit(ip_address: str, provider: str) -> bool:
    """
    Check if IP exceeds hourly usage limit
    
    Returns:
        True if within limit, False if violation
    """
    
    rule = get_provider_rule(provider)
    hourly_limit = rule.hourly_limit
    
    # Count uses in past hour
    one_hour_ago = now() - 3600
    
    usage_count = db.query("""
        SELECT COUNT(*)
        FROM ip_usage_logs
        WHERE device_ip_id IN (
            SELECT id FROM device_ips WHERE ip_address = :ip
        )
        AND provider = :provider
        AND used_at > :one_hour_ago
    """, ip=ip_address, provider=provider, one_hour_ago=one_hour_ago)
    
    return usage_count < hourly_limit
```

**Example:**
```
Rule: hourly_limit = 50
Uses in past hour: 48
Result: OK (48 < 50)
```

#### 4. Team Uniqueness
**Rule:** Each team must use unique IPs (no IP sharing across teams)

```python
def check_team_uniqueness(ip_address: str, team_id: str, provider: str) -> bool:
    """
    Check if other teams are using this IP
    
    Returns:
        True if unique to team, False if used by other teams
    """
    
    rule = get_provider_rule(provider)
    if not rule.require_unique_per_team:
        return True  # Rule not enforced
    
    # Find teams currently using this IP
    teams_using_ip = db.query("""
        SELECT DISTINCT a.team_id
        FROM device_ips dip
        JOIN devices d ON d.id = dip.device_id
        JOIN accounts a ON a.id = d.account_id
        WHERE dip.ip_address = :ip
          AND dip.is_active = TRUE
          AND a.team_id != :team_id
    """, ip=ip_address, team_id=team_id)
    
    return len(teams_using_ip) == 0
```

### Conflict Detection Flow

```python
def detect_conflicts(device_ids: List[str], provider: str) -> List[Conflict]:
    """
    Detect all IP conflicts for given devices
    
    Returns:
        List of conflicts found
    """
    
    conflicts = []
    
    # Group devices by IP
    device_ips = get_device_ips(device_ids)
    ip_groups = group_by_ip(device_ips)
    
    for ip_address, devices in ip_groups.items():
        # Check concurrent limit
        if not check_concurrent_limit(ip_address, provider):
            conflicts.append(Conflict(
                type='concurrent_limit_exceeded',
                ip_address=ip_address,
                devices=devices,
                severity='high'
            ))
        
        # Check cooldown for each device
        for device in devices:
            if not check_cooldown(ip_address, provider):
                conflicts.append(Conflict(
                    type='cooldown_violation',
                    ip_address=ip_address,
                    device_id=device.id,
                    severity='medium'
                ))
        
        # Check hourly limit
        if not check_hourly_limit(ip_address, provider):
            conflicts.append(Conflict(
                type='hourly_limit_exceeded',
                ip_address=ip_address,
                devices=devices,
                severity='high'
            ))
        
        # Check team uniqueness
        for device in devices:
            if not check_team_uniqueness(ip_address, device.team_id, provider):
                conflicts.append(Conflict(
                    type='team_uniqueness_violation',
                    ip_address=ip_address,
                    device_id=device.id,
                    severity='medium'
                ))
    
    return conflicts
```

## Proxy Rotation Algorithm

### Purpose
Automatically rotate devices to healthy proxies when conflicts detected.

### Proxy Selection Criteria

1. **Status**: Must be 'active' (not 'degraded' or 'disabled')
2. **Health Score**: Prefer high health score (>0.7)
3. **Geographic Region**: Match requested region if specified
4. **Not Banned**: Not in banned_by_providers list for this provider
5. **Usage Balance**: Prefer less-used proxies

### Algorithm

```python
def select_proxy(provider: str, preferred_region: str = None) -> Proxy:
    """
    Select best available proxy for rotation
    
    Args:
        provider: Provider name
        preferred_region: Optional geographic preference
    
    Returns:
        Selected proxy or None if none available
    """
    
    # Get candidate proxies
    candidates = db.query("""
        SELECT *
        FROM proxy_pool
        WHERE status = 'active'
          AND health_score > 0.5
          AND NOT (:provider = ANY(banned_by_providers))
        ORDER BY 
            CASE WHEN geographic_region = :region THEN 0 ELSE 1 END,
            health_score DESC,
            total_uses ASC
        LIMIT 10
    """, provider=provider, region=preferred_region)
    
    if not candidates:
        return None
    
    # Weighted random selection (prefer high health score)
    weights = [proxy.health_score ** 2 for proxy in candidates]
    selected = random.choices(candidates, weights=weights, k=1)[0]
    
    return selected
```

### Rotation Flow

```python
def rotate_device_ip(device_id: str, provider: str) -> IPConfig:
    """
    Rotate device to new proxy
    
    Returns:
        New IP configuration
    """
    
    # 1. Select best proxy
    proxy = select_proxy(provider)
    
    if proxy is None:
        raise NoProxyAvailableError()
    
    # 2. Deactivate current IP
    db.execute("""
        UPDATE device_ips
        SET is_active = FALSE, active_until = NOW()
        WHERE device_id = :device_id AND is_active = TRUE
    """, device_id=device_id)
    
    # 3. Create new device_ip record
    new_ip = db.insert("""
        INSERT INTO device_ips (device_id, ip_address, ip_type, proxy_config)
        VALUES (:device_id, :ip, 'proxy', :config)
        RETURNING *
    """, 
        device_id=device_id,
        ip=proxy.ip_address,
        config={
            'proxy_id': proxy.id,
            'port': proxy.port,
            'protocol': proxy.protocol,
            'username': proxy.username,
            'configured_at': now()
        }
    )
    
    # 4. Return configuration for mobile device
    return IPConfig(
        ip_address=proxy.ip_address,
        port=proxy.port,
        protocol=proxy.protocol,
        username=proxy.username,
        password=decrypt(proxy.password_encrypted)
    )
```

### Health Monitoring

```python
def update_proxy_health(proxy_id: str, success: bool):
    """
    Update proxy health score after usage
    
    Args:
        proxy_id: Proxy UUID
        success: Whether usage was successful
    """
    
    proxy = db.get_proxy(proxy_id)
    
    # Increment counters
    proxy.total_uses += 1
    if not success:
        proxy.failed_uses += 1
    
    # Recalculate health score
    proxy.health_score = 1.0 - (proxy.failed_uses / proxy.total_uses)
    
    # Update status based on health
    if proxy.health_score < 0.3:
        proxy.status = 'disabled'
    elif proxy.health_score < 0.5:
        proxy.status = 'degraded'
    else:
        proxy.status = 'active'
    
    proxy.last_health_check = now()
    proxy.save()
```

## Auto-Pause Logic

### Purpose
Automatically pause accounts when balance drops below threshold.

### Trigger Conditions

1. **After Settlement**: Check balance after each round settlement
2. **On Balance Update**: Check after manual balance adjustments
3. **Periodic Check**: Hourly background job

### Algorithm

```python
def check_and_pause_low_balance(account_id: str):
    """
    Check if account should be auto-paused
    
    Auto-pauses if balance < min_balance_threshold
    """
    
    account = db.get_account(account_id)
    capital = db.get_capital(account_id)
    
    # Calculate available balance
    available = capital.balance - capital.locked
    
    # Check threshold
    if available < account.min_balance_threshold:
        # Pause account
        account.status = 'paused'
        account.paused_at = now()
        account.pause_reason = 'low_balance'
        account.save()
        
        # Create audit log
        log_audit(
            actor='system',
            action='account.auto_paused',
            entity_type='Account',
            entity_id=account.id,
            payload={
                'available_balance': available,
                'threshold': account.min_balance_threshold
            }
        )
        
        # Notify admin
        send_notification(
            recipient=get_admins(),
            type='low_balance_alert',
            data={
                'account_id': account.id,
                'balance': available,
                'threshold': account.min_balance_threshold
            }
        )
```

### Re-Enable Logic

```python
def check_unpause_eligibility(account_id: str) -> bool:
    """
    Check if paused account can be re-enabled
    
    Returns:
        True if eligible for unpause
    """
    
    account = db.get_account(account_id)
    
    # Only check auto-paused accounts
    if account.status != 'paused' or account.pause_reason != 'low_balance':
        return False
    
    capital = db.get_capital(account_id)
    available = capital.balance - capital.locked
    
    # Require 110% of threshold for unpause (hysteresis)
    required = account.min_balance_threshold * 1.10
    
    return available >= required
```

## Time Synchronization

### Purpose
Ensure devices execute bets at precise timestamps.

### NTP-Style Synchronization

```python
def synchronize_time(server_url: str, samples: int = 5) -> TimeOffset:
    """
    Synchronize local time with server using multiple samples
    
    Args:
        server_url: Server time endpoint
        samples: Number of samples to take (default 5)
    
    Returns:
        Time offset and precision estimate
    """
    
    offsets = []
    rtts = []
    
    for i in range(samples):
        # Measure offset
        t0 = local_time_ms()
        server_time = http_get(f"{server_url}/time/sync")
        t1 = local_time_ms()
        
        rtt = t1 - t0
        offset = server_time - (t0 + rtt / 2)
        
        offsets.append(offset)
        rtts.append(rtt)
        
        # Small delay between samples
        sleep(0.1)
    
    # Use median offset (filters outliers)
    median_offset = sorted(offsets)[len(offsets) // 2]
    
    # Estimate precision (half of min RTT)
    precision = min(rtts) / 2
    
    return TimeOffset(
        offset_ms=median_offset,
        precision_ms=precision
    )
```

### Sync Schedule

```python
def should_sync_time(last_sync: datetime, next_round: datetime) -> bool:
    """
    Determine if time sync is needed
    
    Returns:
        True if sync recommended
    """
    
    now = current_time()
    
    # Sync if never synced
    if last_sync is None:
        return True
    
    # Sync if last sync > 5 minutes ago
    if (now - last_sync).total_seconds() > 300:
        return True
    
    # Sync if round execution < 10 seconds away
    if next_round and (next_round - now).total_seconds() < 10:
        return True
    
    return False
```

## Capital Management Rules

### Locking Rules

```python
def lock_capital(account_id: str, amount: float, reason: str):
    """
    Lock capital for allocation
    
    Raises:
        InsufficientFundsError if not enough available balance
    """
    
    capital = db.get_capital(account_id)
    available = capital.balance - capital.locked
    
    # Verify sufficient funds
    if available < amount:
        raise InsufficientFundsError(
            f"Insufficient funds: {available} < {amount}"
        )
    
    # Lock funds
    capital.locked += amount
    capital.updated_at = now()
    capital.save()
    
    # Audit log
    log_audit(
        actor='system',
        action='capital.locked',
        entity_type='Capital',
        entity_id=account_id,
        payload={
            'amount': amount,
            'reason': reason,
            'new_locked': capital.locked
        }
    )
```

### Settlement Rules

```python
def settle_allocation(allocation: Allocation, outcome: str, payout: float = 0):
    """
    Settle allocation after round completion
    
    Args:
        allocation: The allocation to settle
        outcome: 'win', 'loss', 'tie', or 'push'
        payout: Amount won (if outcome = 'win')
    """
    
    capital = db.get_capital(allocation.account_id)
    
    # Update balance based on outcome
    if outcome == 'win':
        capital.balance += payout  # Add winnings
    elif outcome == 'loss':
        capital.balance -= allocation.amount  # Deduct bet
    elif outcome in ['tie', 'push']:
        pass  # No change (bet returned)
    
    # Unlock funds
    capital.locked -= allocation.amount
    capital.updated_at = now()
    capital.save()
    
    # Update allocation record
    allocation.outcome = outcome
    allocation.payout = payout if outcome == 'win' else 0
    allocation.save()
    
    # Check auto-pause
    check_and_pause_low_balance(allocation.account_id)
```

## Matchup Balancing

### Purpose
Create fair matchups by balancing team strength.

### Balance Score Calculation

```python
def calculate_balance_score(team_a: Team, team_b: Team) -> float:
    """
    Calculate how balanced a matchup is
    
    Returns:
        Score from 0.0 (unbalanced) to 1.0 (perfectly balanced)
    """
    
    # Get total commission for each team
    commission_a = sum(acc.commission_pct for acc in team_a.accounts)
    commission_b = sum(acc.commission_pct for acc in team_b.accounts)
    
    # Calculate ratio (smaller / larger)
    if commission_a == 0 or commission_b == 0:
        return 0.0
    
    ratio = min(commission_a, commission_b) / max(commission_a, commission_b)
    
    return ratio
```

### Auto-Matchup Selection

```python
def find_balanced_matchup(provider: str) -> Tuple[Team, Team]:
    """
    Find most balanced pair of teams for matchup
    
    Returns:
        Tuple of (team_a, team_b) with best balance score
    """
    
    # Get active teams
    teams = db.query("""
        SELECT * FROM teams
        WHERE status = 'active'
          AND EXISTS (
              SELECT 1 FROM accounts 
              WHERE team_id = teams.id 
                AND status = 'active'
          )
    """)
    
    if len(teams) < 2:
        raise NotEnoughTeamsError()
    
    # Find best pair
    best_pair = None
    best_score = 0
    
    for i, team_a in enumerate(teams):
        for team_b in teams[i+1:]:
            # Don't match same role
            if team_a.role == team_b.role:
                continue
            
            score = calculate_balance_score(team_a, team_b)
            
            if score > best_score:
                best_score = score
                best_pair = (team_a, team_b)
    
    if best_pair is None:
        raise NoValidMatchupError()
    
    return best_pair
```
