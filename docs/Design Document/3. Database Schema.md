## Core Tables

### 1. Teams

**Purpose**: Logical groups of accounts that bet together

```sql
CREATE TABLE teams (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    role VARCHAR(20) NOT NULL CHECK (role IN ('PRIMARY', 'COUNTER')),
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'paused')),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_teams_status (status, created_at),
    INDEX idx_teams_role (role, status)
);
```

**Fields**:
- `id` - UUID primary key
- `name` - Display name (e.g., "Primary Team A")
- `role` - Either PRIMARY (main team) or COUNTER (opposing team)
- `status` - active, inactive, or paused
- `created_at`, `updated_at` - Timestamps

**Example Data**:
```sql
INSERT INTO teams (name, role, status) VALUES
('Team Alpha', 'PRIMARY', 'active'),
('Team Beta', 'COUNTER', 'active'),
('Team Gamma', 'PRIMARY', 'paused');
```

### 2. Accounts

**Purpose**: Individual betting accounts with commission rates

```sql
CREATE TABLE accounts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    provider VARCHAR(50) NOT NULL,
    commission_pct DECIMAL(5,2) NOT NULL CHECK (commission_pct >= 0 AND commission_pct <= 100),
    team_id UUID REFERENCES teams(id) ON DELETE SET NULL,
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'paused')),
    min_balance_threshold DECIMAL(10,2) DEFAULT 100.00,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_accounts_team (team_id, status),
    INDEX idx_accounts_provider (provider, status),
    INDEX idx_accounts_commission (commission_pct DESC)
);
```

**Fields**:
- `id` - UUID primary key
- `provider` - Provider name (e.g., "evolution", "pragmatic")
- `commission_pct` - Commission percentage (0-100)
- `team_id` - Foreign key to teams table
- `status` - active, inactive, or paused
- `min_balance_threshold` - Balance below which account auto-pauses
- `created_at`, `updated_at` - Timestamps

**Example Data**:
```sql
INSERT INTO accounts (provider, commission_pct, team_id, status) VALUES
('evolution', 5.00, 'team_alpha_uuid', 'active'),
('evolution', 3.50, 'team_alpha_uuid', 'active'),
('evolution', 4.20, 'team_beta_uuid', 'active');
```

### 3. Capitals

**Purpose**: Track account balances and locked funds

```sql
CREATE TABLE capitals (
    account_id UUID PRIMARY KEY REFERENCES accounts(id) ON DELETE CASCADE,
    balance DECIMAL(15,2) NOT NULL DEFAULT 0,
    locked DECIMAL(15,2) NOT NULL DEFAULT 0,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Constraints
    CHECK (balance >= 0),
    CHECK (locked >= 0),
    CHECK (balance >= locked),
    
    -- Indexes
    INDEX idx_capitals_balance (balance),
    INDEX idx_capitals_locked (locked)
);
```

**Fields**:
- `account_id` - Primary key and foreign key to accounts
- `balance` - Available + locked funds
- `locked` - Funds reserved for active rounds
- `updated_at` - Last modification timestamp

**Invariants**:
- `balance >= locked` (can't lock more than available)
- Both must be non-negative

**Example Data**:
```sql
INSERT INTO capitals (account_id, balance, locked) VALUES
('account_1_uuid', 1000.00, 150.00),
('account_2_uuid', 750.00, 0.00),
('account_3_uuid', 500.00, 75.00);
```

### 4. Devices

**Purpose**: Mobile devices executing bets

```sql
CREATE TABLE devices (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100),
    android_id VARCHAR(100) UNIQUE,
    auth_token TEXT,
    account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,
    status VARCHAR(20) DEFAULT 'offline' CHECK (status IN ('online', 'offline', 'error')),
    last_seen TIMESTAMP,
    battery_level INT CHECK (battery_level >= 0 AND battery_level <= 100),
    app_version VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_devices_status (status, last_seen),
    INDEX idx_devices_account (account_id)
);
```

**Fields**:
- `id` - UUID primary key
- `name` - User-friendly name (e.g., "Pixel 7 Pro #1")
- `android_id` - Unique device identifier
- `auth_token` - JWT token for authentication
- `account_id` - Which account this device operates
- `status` - online, offline, or error
- `last_seen` - Last heartbeat timestamp
- `battery_level` - Current battery percentage
- `app_version` - Mobile app version (e.g., "1.0.0")

**Example Data**:

### 5. Matchups

**Purpose**: Pair two opposing teams for betting rounds

```sql
CREATE TABLE matchups (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    provider VARCHAR(50) NOT NULL,
    table_id VARCHAR(100) NOT NULL,
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'paused', 'completed')),
    locked_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_matchups_provider (provider, status),
    INDEX idx_matchups_table (table_id, status),
    INDEX idx_matchups_status (status, created_at)
);
```

**Fields**:
- `id` - UUID primary key
- `provider` - Provider name (e.g., "evolution")
- `table_id` - Specific table/room identifier
- `status` - active, paused, or completed
- `locked_at` - Timestamp when locked (prevents changes during active rounds)
- `created_at`, `updated_at` - Timestamps

**Example Data**:
```sql
INSERT INTO matchups (provider, table_id, status) VALUES
('evolution', 'baccarat_001', 'active'),
('evolution', 'baccarat_002', 'active'),
('pragmatic', 'main_table', 'paused');
```

### 6. Matchup_Teams (Junction Table)

**Purpose**: Associate teams with matchups

```sql
CREATE TABLE matchup_teams (
    matchup_id UUID REFERENCES matchups(id) ON DELETE CASCADE,
    team_id UUID REFERENCES teams(id) ON DELETE CASCADE,
    side VARCHAR(20) NOT NULL CHECK (side IN ('banker', 'player', 'tie')),
    PRIMARY KEY (matchup_id, team_id),
    
    -- Indexes
    INDEX idx_matchup_teams_matchup (matchup_id),
    INDEX idx_matchup_teams_team (team_id)
);
```

**Fields**:
- `matchup_id` - Foreign key to matchups
- `team_id` - Foreign key to teams
- `side` - Which side this team bets on (banker, player, or tie)

**Constraints**:
- Each matchup can have multiple teams (1v1, 1v2, etc.)
- Each team can only be in matchup once per matchup
- Side determines betting strategy

**Example Data**:
```sql
INSERT INTO matchup_teams (matchup_id, team_id, side) VALUES
('matchup_1_uuid', 'team_alpha_uuid', 'banker'),
('matchup_1_uuid', 'team_beta_uuid', 'player');
```

### 7. Rounds

**Purpose**: Individual betting rounds with execution timing

```sql
CREATE TABLE rounds (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    matchup_id UUID REFERENCES matchups(id) ON DELETE CASCADE,
    execute_at BIGINT NOT NULL,
    status VARCHAR(20) DEFAULT 'prepared' CHECK (status IN ('preparing', 'prepared', 'executing', 'completed', 'aborted')),
    seed VARCHAR(64) NOT NULL,
    total_capital DECIMAL(15,2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_rounds_execute (execute_at, status),
    INDEX idx_rounds_matchup (matchup_id, status),
    INDEX idx_rounds_status (status, created_at)
);
```

**Fields**:
- `id` - UUID primary key
- `matchup_id` - Foreign key to matchups
- `execute_at` - Unix timestamp in milliseconds when bets should be placed
- `status` - Round state (preparing → prepared → executing → completed/aborted)
- `seed` - Random seed for reproducible allocation
- `total_capital` - Total amount to distribute
- `created_at`, `updated_at` - Timestamps

**State Transitions**:
```
preparing → prepared → executing → completed
    ↓           ↓          ↓
  aborted    aborted    aborted
```

**Example Data**:
```sql
INSERT INTO rounds (matchup_id, execute_at, status, seed, total_capital) VALUES
('matchup_1_uuid', 1706400000000, 'prepared', 'seed_12345', 1000.00),
('matchup_1_uuid', 1706400030000, 'executing', 'seed_67890', 1200.00);
```

### 8. Allocations

**Purpose**: Specific bet assignments per account per round

```sql
CREATE TABLE allocations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    round_id UUID REFERENCES rounds(id) ON DELETE CASCADE,
    account_id UUID REFERENCES accounts(id) ON DELETE CASCADE,
    side VARCHAR(20) NOT NULL CHECK (side IN ('banker', 'player', 'tie')),
    amount DECIMAL(10,2) NOT NULL CHECK (amount > 0),
    outcome VARCHAR(20) CHECK (outcome IN ('win', 'loss', 'tie', 'push', NULL)),
    payout DECIMAL(10,2),
    executed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Constraints
    UNIQUE(round_id, account_id),
    
    -- Indexes
    INDEX idx_allocations_round (round_id),
    INDEX idx_allocations_account (account_id, created_at),
    INDEX idx_allocations_outcome (outcome)
);
```

**Fields**:
- `id` - UUID primary key
- `round_id` - Foreign key to rounds
- `account_id` - Foreign key to accounts
- `side` - banker, player, or tie
- `amount` - Bet amount (must be positive)
- `outcome` - win, loss, tie, push, or NULL (if not yet settled)
- `payout` - Amount won (if outcome = 'win')
- `executed_at` - Timestamp when bet was placed
- `created_at` - When allocation was created

**Example Data**:
```sql
INSERT INTO allocations (round_id, account_id, side, amount, outcome, payout) VALUES
('round_1_uuid', 'account_1_uuid', 'banker', 50.00, 'win', 95.00),
('round_1_uuid', 'account_2_uuid', 'banker', 30.00, 'win', 57.00),
('round_1_uuid', 'account_3_uuid', 'player', 40.00, 'loss', 0.00);
```

### 11. IP_Conflict_Rules

**Purpose**: Provider-specific IP usage rules

```sql
CREATE TABLE ip_conflict_rules (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    cooldown_seconds INT NOT NULL DEFAULT 300,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

```

**Fields**:

- `provider` - Provider name
- `max_concurrent_devices` - Max devices using same IP simultaneously
- `cooldown_seconds` - Time before IP can be reused after last use
- `hourly_limit` - Max uses of same IP per hour
- `require_unique_per_team` - If true, each team must use unique IPs

### 12. Proxy_Pool

**Purpose**: Manage available proxy servers

```sql
CREATE TABLE proxy_pool (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    ip_address INET NOT NULL UNIQUE,
    port INT NOT NULL,
    protocol VARCHAR(20) NOT NULL CHECK (protocol IN ('http', 'https', 'socks4', 'socks5')),
    username VARCHAR(255),
    password_encrypted TEXT,
    geographic_region VARCHAR(50),
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'degraded', 'disabled')),
    health_score DECIMAL(3,2) DEFAULT 1.00 CHECK (health_score >= 0 AND health_score <= 1),
    total_uses INT DEFAULT 0,
    failed_uses INT DEFAULT 0,
    banned_by_providers TEXT[],
    last_health_check TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes
    INDEX idx_proxy_status (status, health_score),
    INDEX idx_proxy_region (geographic_region, status),
    INDEX idx_proxy_health (health_score DESC)
);
```

**Fields**:
- `id` - UUID primary key
- `ip_address` - Proxy IP address
- `port` - Proxy port
- `protocol` - http, https, socks4, or socks5
- `username`, `password_encrypted` - Auth credentials
- `geographic_region` - E.g., "US-East", "EU-West"
- `status` - active, degraded, or disabled
- `health_score` - 0.0 to 1.0 (calculated from success rate)
- `total_uses`, `failed_uses` - Usage counters
- `banned_by_providers` - Array of providers that banned this IP
- `last_health_check` - Last health check timestamp

**Health Score Calculation**:
```
health_score = 1.0 - (failed_uses / total_uses)

if health_score < 0.5: status = 'degraded'
if health_score < 0.3: status = 'disabled'
```

**Example Data**:
```sql
INSERT INTO proxy_pool (ip_address, port, protocol, geographic_region, health_score) VALUES
('203.0.113.1', 8080, 'socks5', 'US-East', 0.95),
('203.0.113.2', 8080, 'socks5', 'EU-West', 0.88),
('203.0.113.3', 8080, 'http', 'AS-Southeast', 0.42);
```

## Database Constraints Summary

### Primary Key Strategy
- All tables use UUID primary keys generated by PostgreSQL
- UUIDs prevent ID collision in distributed systems
- Generated via `gen_random_uuid()`

### Foreign Key Strategy
- All foreign keys use `ON DELETE` clauses
- `CASCADE` - Delete child records when parent deleted
- `SET NULL` - Set FK to NULL when parent deleted
- `RESTRICT` - Prevent deletion if children exist

### Check Constraints
- Enforce data validity at database level
- Examples:
  - `balance >= locked`
  - `commission_pct >= 0 AND commission_pct <= 100`
  - `status IN ('active', 'inactive', 'paused')`

### Index Strategy
- Foreign keys automatically indexed
- Composite indexes for common query patterns
- Partial indexes for filtered queries
- `created_at DESC` for time-series queries

## Common Queries

### Get Active Accounts for a Team
```sql
SELECT a.id, a.provider, a.commission_pct, c.balance
FROM accounts a
JOIN capitals c ON c.account_id = a.id
WHERE a.team_id = :team_id
  AND a.status = 'active'
  AND c.balance > a.min_balance_threshold
ORDER BY a.commission_pct DESC;
```

### Get Allocations for a Round
```sql
SELECT 
    al.id,
    al.side,
    al.amount,
    a.provider AS account_provider,
    d.name AS device_name
FROM allocations al
JOIN accounts a ON a.id = al.account_id
JOIN devices d ON d.account_id = a.id
WHERE al.round_id = :round_id
ORDER BY al.amount DESC;
```

### Check IP Conflicts
```sql
-- Concurrent usage check
SELECT ip_address, COUNT(*) as device_count
FROM device_ips
WHERE is_active = TRUE
GROUP BY ip_address
HAVING COUNT(*) > (
    SELECT max_concurrent_devices 
    FROM ip_conflict_rules 
    WHERE provider = :provider
);

-- Cooldown check
SELECT dip.ip_address, MAX(iul.used_at) as last_used
FROM device_ips dip
JOIN ip_usage_logs iul ON iul.device_ip_id = dip.id
WHERE iul.provider = :provider
  AND iul.used_at > NOW() - INTERVAL '5 minutes'
GROUP BY dip.ip_address;
```

### Proxy Health Report
```sql
SELECT 
    ip_address,
    port,
    geographic_region,
    status,
    health_score,
    total_uses,
    failed_uses,
    ROUND((failed_uses::DECIMAL / NULLIF(total_uses, 0)) * 100, 2) as failure_rate_pct
FROM proxy_pool
WHERE status != 'disabled'
ORDER BY health_score DESC;
```

## Performance Optimization

### Query Optimization
- Use `EXPLAIN ANALYZE` for slow queries
- Add indexes for WHERE, JOIN, ORDER BY columns
- Avoid SELECT * (specify needed columns)
- Use prepared statements to prevent SQL injection

### Partitioning Strategy
- Partition `audit_logs` by month
- Partition `telemetry` by month
- Consider partitioning `allocations` if > 10M rows

### Maintenance Tasks
```sql
-- Vacuum analyze (run weekly)
VACUUM ANALYZE;

-- Reindex (run monthly)
REINDEX DATABASE betting_coordinator;

-- Update statistics (run daily)
ANALYZE;
```

## Backup Strategy

### Daily Backups
```bash
pg_dump -U postgres -d betting_coordinator \
  --format=custom \
  --file=backup_$(date +%Y%m%d).dump
```

### Point-in-Time Recovery
- Enable WAL archiving
- Retain 7 days of WAL logs
- Test restore monthly
