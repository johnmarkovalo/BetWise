## System Architecture

### Overview
The system follows a **centralized planning, distributed execution** architecture:
- **Backend**: Makes all decisions (what, when, how much)
- **Mobile Devices**: Execute instructions locally with precision timing

This separation ensures:
- Consistency (single source of truth)
- Scalability (backend handles all complexity)
- Reliability (devices can operate semi-autonomously)

#### 3. Allocation Engine
**Purpose**: Calculate bet amounts per account

**Components**:
- `AllocationEngine` - Core algorithm
- `SeededRandom` - Reproducible randomness
- `CapitalValidator` - Balance checks

**Algorithm**:
```php
function allocate(accounts, totalCapital, seed):
    rng = SeededRandom(seed)
    
    // Step 1: Calculate weights
    totalCommission = sum(account.commission for account in accounts)
    weights = [account.commission / totalCommission for account in accounts]
    
    // Step 2: Base allocation
    baseAmounts = [totalCapital * weight for weight in weights]
    
    // Step 3: Apply randomization
    randomizedAmounts = []
    for baseAmount in baseAmounts:
        variance = baseAmount * 0.15
        offset = rng.float(-variance, variance)
        randomizedAmounts.append(baseAmount + offset)
    
    // Step 4: Enforce monotonicity
    sortedByCommission = sortWithCommission(randomizedAmounts)
    previous = 0
    finalAmounts = []
    for amount in sortedByCommission:
        final = max(amount, previous + 0.01)
        finalAmounts.append(final)
        previous = final
    
    // Step 5: Normalize to total
    scale = totalCapital / sum(finalAmounts)
    return [amount * scale for amount in finalAmounts]
```

**Guarantees**:
- Commission-weighted (fair distribution)
- Monotonic (higher commission ≥ higher bet)
- Randomized (unpredictable patterns)
- Capital-respecting (never exceeds budget)
- Reproducible (same seed = same result)

#### 5. Round Lifecycle Manager
**Purpose**: Orchestrate round state transitions

**State Machine**:
```
PREPARING → PREPARED → EXECUTING → COMPLETED
     ↓           ↓          ↓
   ABORTED    ABORTED    ABORTED
```

**Transitions**:
- `PREPARING`: Running allocation, locking capital
- `PREPARED`: Instructions broadcast, devices acknowledged
- `EXECUTING`: Devices executing bets
- `COMPLETED`: Results collected, balances updated
- `ABORTED`: Emergency stop triggered

**Components**:
- `RoundService` - State transitions
- `Round` model - State machine implementation
- `RoundObserver` - Event listeners

#### 6. Capital Management
**Purpose**: Track funds, prevent over-betting

**Components**:
- `CapitalService` - Balance operations
- `Capital` model - Account balances
- `CapitalSnapshot` - Historical records

**Operations**:
```php
// Lock capital for round
function lockCapital(accountId, amount):
    capital = Capital::find(accountId)
    if capital.balance - capital.locked < amount:
        throw InsufficientFundsException
    capital.locked += amount
    capital.save()

// Settle after round
function settle(allocation, outcome):
    capital = Capital::find(allocation.account_id)
    
    if outcome == 'win':
        capital.balance += allocation.payout
    elif outcome == 'loss':
        capital.balance -= allocation.amount
    
    capital.locked -= allocation.amount
    capital.save()
```

**Auto-Pause Logic**:
```php
function checkAutoPause(account):
    capital = account.capital
    threshold = account.min_balance_threshold
    
    if capital.balance < threshold:
        account.status = 'paused'
        account.save()
        
        event(AccountAutoPaused(account))
        
        // Notify admin
        Notification::send(
            Admin::all(),
            new LowBalanceAlert(account)
        )
```

#### 3. Time Synchronization Manager
**Purpose**: Accurate server time matching

**Algorithm**:
```kotlin
class TimeSyncManager(private val api: ApiClient) {
    private var offset: Long = 0
    
    suspend fun sync(): Long {
        val samples = List(5) { measureOffset() }
        // Use median to filter outliers
        offset = samples.sorted()[samples.size / 2]
        return offset
    }
    
    private suspend fun measureOffset(): Long {
        val t0 = System.currentTimeMillis()
        val serverTime = api.getServerTime()
        val t1 = System.currentTimeMillis()
        val rtt = t1 - t0
        
        // Server time when request left client
        return serverTime - (t0 + rtt / 2)
    }
    
    fun getServerTime(): Long = 
        System.currentTimeMillis() + offset
}
```

**Sync Schedule**:
- Initial: On app launch
- Periodic: Every 5 minutes
- Pre-round: 10 seconds before execution
- Recovery: After network reconnection

#### 4. Round Scheduler
**Purpose**: Execute bets at precise timestamp

**Implementation**:
```kotlin
class RoundScheduler(
    private val timeSyncManager: TimeSyncManager,
    private val gestureExecutor: GestureExecutor
) {
    suspend fun scheduleRound(round: RoundInstruction) {
        val now = timeSyncManager.getServerTime()
        val delay = round.executeAt - now
        
        if (delay > 0) {
            delay(delay)
        }
        
        // Execute bet
        gestureExecutor.executeBet(
            side = round.side,
            amount = round.amount
        )
    }
}
```

#### 5. Provider Adapter Pattern (NEW)
**Purpose**: Abstract provider-specific UI logic

**Interface**:
```kotlin
interface ProviderAdapter {
    suspend fun detectBettingWindow(): Boolean
    suspend fun placeBet(side: String, amount: Double): Result<Unit>
    suspend fun detectResult(): BetOutcome
    fun getProviderName(): String
}

// Evolution Gaming implementation
class EvolutionGamingAdapter(
    private val gestureExecutor: GestureExecutor,
    private val nodeF finder: UiNodeFinder
) : ProviderAdapter {
    
    override suspend fun detectBettingWindow(): Boolean {
        val root = accessibilityService.rootInActiveWindow
        
        // Look for "PLACE YOUR BETS" indicator
        val indicator = nodeFinder.findByText(
            root, 
            "PLACE YOUR BETS"
        )
        
        return indicator != null && indicator.isVisible
    }
    
    override suspend fun placeBet(
        side: String, 
        amount: Double
    ): Result<Unit> {
        return runCatching {
            // 1. Find bet button
            val betButton = when (side) {
                "banker" -> findBankerButton()
                "player" -> findPlayerButton()
                "tie" -> findTieButton()
                else -> throw UnknownSideException(side)
            }
            
            // 2. Tap button
            gestureExecutor.tap(betButton)
            delay(300)
            
            // 3. Enter amount
            val amountField = findAmountField()
            gestureExecutor.enterText(amountField, amount)
            delay(200)
            
            // 4. Confirm
            val confirmButton = findConfirmButton()
            gestureExecutor.tap(confirmButton)
        }
    }
    
    override suspend fun detectResult(): BetOutcome {
        val root = accessibilityService.rootInActiveWindow
        
        // Look for result display
        val resultNode = nodeFinder.findByClassName(
            root,
            "ResultDisplay"
        )
        
        val resultText = resultNode?.text?.toString()
        
        return when {
            resultText?.contains("BANKER") == true -> 
                BetOutcome.BANKER_WIN
            resultText?.contains("PLAYER") == true -> 
                BetOutcome.PLAYER_WIN
            resultText?.contains("TIE") == true -> 
                BetOutcome.TIE
            else -> BetOutcome.UNKNOWN
        }
    }
}
```

**Adapter Registry**:
```kotlin
object ProviderAdapterRegistry {

    fun register(provider: String, adapter: ProviderAdapter) {
        adapters[provider] = adapter
    }
    
    fun get(provider: String): ProviderAdapter? = 
        adapters[provider]

// Usage

class GestureExecutor(
    private val service: AccessibilityService
) {
    suspend fun tap(node: AccessibilityNodeInfo): Result<Unit> = 
        suspendCancellableCoroutine { continuation ->
            val rect = Rect()
            node.getBoundsInScreen(rect)

            val gesture = GestureDescription.Builder()

                .build()

                        continuation.resume(Result.success(Unit))
                    }

                        continuation.resume(
                            Result.failure(GestureCancelledException())
                        )
                    }
                },
                null
            )
        }

## Data Flow

### Round Execution Flow

```
┌──────────────┐
│ 1. PREPARE   │
│   (Backend)  │
└──────┬───────┘
       │
       ├─> Run allocation engine
       ├─> Check IP conflicts
       ├─> Rotate proxies if needed
       ├─> Lock capital
       └─> Generate round ID + seed
       │
       ↓
┌──────────────┐
│ 2. BROADCAST │
│   (Reverb)   │
└──────┬───────┘
       │
       ├─> Send to device-specific channels
       ├─> Include HMAC signature
       └─> Wait for acknowledgments
       │
       ↓
┌──────────────┐
│ 3. SCHEDULE  │
│   (Mobile)   │
└──────┬───────┘
       │
       ├─> Verify signature
       ├─> Schedule execution
       ├─> Send acknowledgment
       └─> Sync time (if needed)
       │
       ↓
┌──────────────┐
│ 4. WAIT      │
│   (Mobile)   │
└──────┬───────┘
       │
       ├─> Monitor betting window
       ├─> Wait for execute_at timestamp
       └─> Prepare gesture
       │
       ↓
┌──────────────┐
│ 5. EXECUTE   │
│   (Mobile)   │
└──────┬───────┘
       │
       ├─> Inject touch gestures
       ├─> Verify bet placed
       └─> Detect game outcome
       │
       ↓
┌──────────────┐
│ 6. REPORT    │
│   (Mobile)   │
└──────┬───────┘
       │
       └─> Send result to backend
       │
       ↓
┌──────────────┐
│ 7. SETTLE    │
│   (Backend)  │
└──────┬───────┘
       │
       ├─> Update balances
       ├─> Unlock capital
       ├─> Check auto-pause
       ├─> Log to audit trail
       └─> Update statistics

## Communication Protocols

### WebSocket Message Format

**Standard Envelope**:
```json
{
  "type": "event_name",
  "timestamp": 1706400000000,
  "payload": { /* event-specific data */ },
  "signature": "hmac_sha256_hash"
}
```

**Signature Calculation**:
```php
$payload = json_encode($message);
$signature = hash_hmac('sha256', $payload, env('HMAC_KEY'));
```

**Message Types**:
1. `round.prepared` - Server sends bet instructions
2. `round.acknowledged` - Device confirms receipt
3. `round.result` - Device reports outcome
4. `round.aborted` - Server cancels round
5. `heartbeat` - Device status ping
6. `time.sync` - Time synchronization request/response

## Security Architecture

### Authentication & Authorization

**Device Authentication**:
```
1. Device generates key pair (public/private)
2. Admin approves device registration
3. Backend issues JWT token (24h expiry)
4. Device uses JWT for all API/WebSocket requests
5. Token refresh via dedicated endpoint
```

**Admin Authentication**:
```
1. Username/password login
2. 2FA required for production
3. Session-based with CSRF protection
4. Role-based access control (RBAC)
```

### Data Security

**At Rest**:
- PostgreSQL encryption (TDE)
- Redis password protection
- Sensitive fields encrypted (proxy passwords)

**In Transit**:
- TLS 1.3 for all connections
- WebSocket over WSS (TLS)
- Certificate pinning on mobile

**Audit Trail**:
- All mutations logged
- IP address tracking
- Timestamp with timezone
- Actor identification (user/system/device)

## Scalability Considerations

### Horizontal Scaling

**Backend**:
- Stateless application servers (Laravel)
- Load balancer (Nginx)
- Shared PostgreSQL (master-replica)
- Redis cluster for cache/sessions
- Multiple Reverb servers with sticky sessions

**Mobile**:
- Unlimited devices (each independent)
- Group devices by provider/table for channel efficiency

### Vertical Scaling

**Database Optimizations**:
- Indexed foreign keys
- Partitioning for audit logs (by month)
- Connection pooling (pgBouncer)
- Read replicas for analytics

**Cache Strategy**:
- Account data (1 hour TTL)
- Team compositions (until mutation)
- Round status (real-time invalidation)
